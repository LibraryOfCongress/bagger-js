Instructions & Limitations

Browser Compatibility

All HTML5 browsers are supported for single-file operations. Unfortunately, support for selecting directories is currently limited to Safari and Google Chrome until the APIs are completely implemented in other browsers. See issue #1 for more information.

Amazon S3 Configuration

CORS

All modern browsers apply CORS restrictions to requests made by our JavaScript. This means that an S3 bucket must have a CORS policy defined following the instructions in the S3 CORS configuration section of the AWS SDK for JavaScript documentation.

The configuration below is known to work: 

<?xml version="1.0" encoding="UTF-8"?>
<CORSConfiguration xmlns="http://s3.amazonaws.com/doc/2006-03-01/">
    <CORSRule>
        <AllowedOrigin>*</AllowedOrigin>
        <AllowedMethod>GET</AllowedMethod>
        <AllowedMethod>PUT</AllowedMethod>
        <AllowedMethod>POST</AllowedMethod>
        <AllowedMethod>DELETE</AllowedMethod>
        <AllowedMethod>HEAD</AllowedMethod>
        <MaxAgeSeconds>3000</MaxAgeSeconds>
        <ExposeHeader>ETag</ExposeHeader>
        <AllowedHeader>*</AllowedHeader>
        <AllowedHeader>x-amz-*</AllowedHeader>
    </CORSRule>
</CORSConfiguration>
